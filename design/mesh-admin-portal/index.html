MESH Admin Portal 2.0

To go along with the relaunch of MESH 2.0, administrators needed a functional new home to manage the platform. So I redesigned the Admin Portal from the ground up, along with a simple but comprehensive new design system.

When it was decided that MESH would relaunch in 2025 with a new, more stable foundation, the core user-facing app wasn‚Äôt the only thing that needed a big update. For years, administrators were using a backend platform built atop third-party libraries. Feedback was that it was slow and not at all intuitive, and I was assigned to fix the latter issue.

The team wanted something functional and lean. Make it clear and simple to use, built upon the same design principles I‚Äôd used for the MESH app ‚Äî but it needed to be light and quick to implement, as the bulk of the dev team‚Äôs time needed to stay on the consumer app.

Design system #

As I started exploring designs for some of the portal‚Äôs main pages, I began to build out a design system. I created a straightforward colour palette accented by our brand‚Äôs primary dark blue with functional additions; specified a collection of necessary typographic styles; and curated a set of icons.

I then carefully selected the most important controls ‚Äî like buttons, menus, dropdowns, and tags ‚Äî that I knew would be needed across the platform for a component library.1 It was important not to sacrifice visual affordance, so subtle depth is used to cue interactivity.

The design system needed to be comprehensive but relatively simple to implement. I carefully selected the most important components, and pared them down without sacrificing visual affordance.

I also settled on a basic visual hierarchy and principles. At the top is a dark header with the site name and a logout option. The bulk of the page is filled by the flexible main content area, with a lower-contrast sidebar to the left providing persistent navigation, and a toolbar above ‚Äî a very familiar desktop paradigm. Alternating row colours make table views easy to parse, and inset tabbed views should be immediately intuitive.

I pulled from classic desktop design for a structure that felt familiar and intuitive, using layout and contrast to help direct attention.

Takeaways #

The admin portal was a massive undertaking, created simultaneously with the MESH 2.0 relaunch. It was my first time touching, or even seeing, the platform, and it was a great opportunity for me to focus on desktop design for a change. And it was a fun challenge creating a design system that was distinct from that of the MESH app but still rooted in our brand guidelines, rich in visual affordance, with clear hierarchy ‚Äî and could be quickly and easily built by our dev team.

We worked closely with the admin team to understand and assess their needs for the portal, delivering a much improved experience for managing all the content that makes MESH what it is. ‚òº
