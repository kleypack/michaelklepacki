MESH 2.0 Design System

I created a comprehensive design system to ease design and development of MESH while addressing feedback to improve the user experience.

Near the end of 2023 it was decided that MESH would be rebuilt from the ground up to remedy years of tech debt and give the platform a new, more solid foundation. When I heard the news I knew it was the perfect opportunity to create and implement a new design system — this could speed up both my work designing the app and the dev team’s work building it.

I could also further refine the visual direction from our 2022 redesign to address user feedback. Many of them are fairly new to digital interfaces, and while they loved the look, some were struggling to actually use parts of the app. This would be a great time to enhance the user experience by bringing added affordance, structural clarity, and consistency to the visual design.

Design system #

With the previous redesign, I created a small style guide and component library which served us well. But this time I was ready to take things further and introduce a complete design system.

I began by specifying a collection of tokens, which would double as CSS variables for the development team. These included everything from colour values to font sizes to numerical values for sizing and spacing.

In practice, this means less hand selecting and hard coding of frequently used values in favour of specifying tokens. It also helps ensure harmonic consistency throughout the app — the sizing and spacing scales help create visual rhythm, as do the typographic styles that are built upon a responsive scale. A robust token collection also gives us flexibility to make changes as needed, as altering a single token instantly updates all components across the platform that utilises it.

In practice, this means less hand selecting and hard coding of frequently used values and helps ensure harmonic consistency

I started with MESH’s brand identity: rooting the app in shades of our primary MESH blue, accented by bold raspberry. Colours and their combinations were carefully selected to ensure AA WCAG accessibility. There are tokens for border radii so items nest beautifully, and tokens for background blurs to emphasise depth within the interface. Tokens for font size, weight, features, and line heights can be combined to form more complex font styles.

Design tokens would lend the app consistency and rhythm while significantly increasing the efficiency of our work.

I again curated a set of icons from Microsoft’s robust Fluent collection. Separate lists of different sizes cover every use case from the smallest instances to large, multicoloured statement moments. The Fluent system also has wonderfully documented creation guidelines, which allows me to create my own icons when needed that fit seamlessly into the larger set.

Microsoft’s Fluent iconography lend a professional but approachable tone to MESH. I maintain a curated collection for us to use and even make my own custom icons, like the ones above in raspberry.

Utilising the tokens and iconography, I created and managed a comprehensive component library. Similar to the tokens, instead of having to build things from scratch (or copy and paste) every time we need a common asset, we can simply call the component and specify its variables1.

This time around, I designed everything with affordance and tactility in mind. Some of MESH’s users struggle to understand flat, minimal design, so I looked to the earlier days of both desktop and mobile computing for inspiration2. Buttons and cards are slightly raised from the background to invite interaction; input fields are inset, ready to be filled; and the whole interface feels lit, featuring edge highlights, subtle gradients, and even some glowing elements.

A sample of components and some of their variable configurations.

Components have set maximum widths and responsive sizes with locked aspect ratios. I also specified responsive breakpoints and created adaptive layouts for larger screens. All of this allows the MESH app to fluidly scale to effectively utilise screen space while keeping things consistent.

The MESH app scales fluidly and dynamically utilises extra space on larger screens.

Rounding out the design system is a changelog which keeps everyone up to date. Changes are tagged as new features, housekeeping, or documentation and can be marked as ready, in-progress, or pre-release.

The changelog keeps everyone on the same page.

Takeaways #

The design system greatly improved our workflow internally. It certainly sped up my work within Figma, allowing me to easily add common components to new designs and select their appropriate variants. Tokens, too, allow me to build quickly while keeping things consistent.

It also vastly sped up our rebuild, which could have been a far greater undertaking without those tokens and components. The front-end developer took time to build out the system first, speeding up their subsequent build of all the individual pages and flows.

And finally, it’s been a hit with users, with excellent response to the new look and speed of the app. ☼
